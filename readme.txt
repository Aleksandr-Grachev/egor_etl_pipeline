2.2 ЗАДАНИЕ (2.2_task)
В данном задании нужно добавить в таблицы схемы 'rd' информацию из csv которой нет в самих таблицах и перегрузить снова информацию в витрину 'LOAN_HOLIDAY_INFO'
• sources - CSV файлы для дополнения информации
• analytics_select.sql - Аналитические запросы, которые помогают понять какие данные уже есть в слое 'rd', а какие нужно добавить из csv
• import_rd.py - py скрипт который импортирует данные из csv в слой 'rd'
• import_rd_with_delete_full_dublicate.py - py скрипт который импортирует данные из csv в слой rd,
   при этом не добавляя полные дубликаты записей из product(полный дубликат - мусор)
• reload_data.sql - SQL скрипт который перегружает данные из слоя 'rd' в витрину 'LOAN_HOLIDAY_INFO'
• description - Описание задачи и подхода к задаче

ПОРЯДОК ВЫПОЛНЕНИЯ СКРИПТОВ:
1) import_rd.py или import_rd_with_delete_full_dublicate.py
2) reload_data.sql

2.3 ЗАДАНИЕ (2.3_task)
В данном задании нам нужно дополнить таблицу dm.dict_currency из csv файла,
исправить значения account_in_sum текущего дня на account_out_sum предыдущего дня или account_out_sum текущего дня на account_in_sum следующего дня
в таблице rd.account_balance, предварительно написав запрос который вычисляет нужное значение для замены
И в конце написать процедуру перегрузки информации из источников данных в конечную витрину

• dict_currency.csv - полный источник данных таблицы dm.dict_currency
• import_dict_currency.py - py скрипт Очистка таблицы dm.dict_currency и импортирование в неё новые данные из csv файла
• update_wrong_amount.sql - SQL скрипт изменения значений account_in_sum текущего дня на account_out_sum предыдущего дня ИЛИ account_out_sum текущего дня на account_in_sum следующего дня
   в таблице rd.account_balance, а так же запрос для вычисления нужных для замены значений
• reload_account_balance_turnover.sql - SQL скрипт перегрузки данных из источников данных в конечную витрину


ПОРЯДОК ВЫПОЛНЕНИЯ СКРИПТОВ:
1) import_dict_currency.py
2) update_wrong_amount.sql
3) reload_account_balance_turnover.sql
